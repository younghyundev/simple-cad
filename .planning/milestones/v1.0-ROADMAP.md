# Milestone v1.0: SimpleCAD MVP

**Status:** SHIPPED 2026-04-26
**Phases:** 1-7
**Total Plans:** 3 formal GSD plans plus direct implementation commits for Phases 1-6

## Overview

v1.0 delivers SimpleCAD as a usable browser-based 2D CAD editor. It supports core drawing and editing, layer/property/history workflows, JSON/SVG/DXF file flows, DWG conversion API wiring, workspace tabs, and cross-tab reference copy/paste.

## Phases

### Phase 1: Editor Foundation

**Goal:** 앱 셸, 도면 모델, 캔버스 좌표계, 그리드, 줌/팬을 만든다.

Delivered:
- Full-screen editor shell
- Canvas rendering and viewport transforms
- Grid, zoom, pan, status bar coordinates
- Internal `CadDocument` model

### Phase 2: Basic Entity Editing

**Goal:** 기본 2D 엔티티 생성, 선택, 이동, 삭제를 구현한다.

Delivered:
- Line, rectangle, circle, polyline, text creation
- Selection and movement
- Delete/erase workflows
- Inline text creation/editing

### Phase 3: Properties, History, Layers

**Goal:** 속성 편집, undo/redo, 레이어 관리를 구현한다.

Delivered:
- Stroke color, width, style editing
- Undo/redo history
- Layer add/rename/color/visibility/lock
- Object layer assignment

### Phase 4: JSON, SVG, DXF

**Goal:** 저장/불러오기, 자동 저장, SVG 내보내기, DXF 입출력 구조를 구현한다.

Delivered:
- JSON open/save
- SVG export
- DXF import/export
- Auto-save
- DXF layer/color/line style/text/curve/block/dimension coverage improvements

### Phase 5: DWG Conversion Workflow

**Goal:** DWG 서버 변환 API, 변환 경고, 어댑터 구조를 구현한다.

Delivered:
- Conversion API client
- Development mock CAD conversion routes
- DWG import/export UI route
- Conversion warning and unsupported-entity surfacing
- Backend API contract documentation

### Phase 6: Workspace UX

**Goal:** 시작 안내 페이지, 최근 열기, 파일 열기, 다중 도면 탭 작업공간을 구현한다.

Delivered:
- Start page with new/open/recent actions
- Recent document persistence
- Multi-document tabs
- Per-tab viewport, selection, and undo/redo history
- Large drawing interaction performance improvements

### Phase 7: Cross-tab Reference Copy and Paste

**Goal:** 탭 간 객체 복사/붙여넣기, 우클릭 컨텍스트 메뉴, 참조점 기반 복사/붙여넣기를 구현한다.

Formal plans:
- [x] 07-01: Clipboard Core
- [x] 07-02: Context Menu and Shortcuts
- [x] 07-03: Reference Point Paste

Delivered:
- CAD clipboard helpers
- Right-click context menu
- Ctrl/Cmd+C and Ctrl/Cmd+V guarded shortcuts
- Cross-tab paste
- Reference copy based on another object's snap point
- Reference paste overlay preview
- Reference clipboard routing through normal paste
- Human UAT 6/6 passed

## Milestone Summary

## Key Decisions

- Use React + TypeScript + Vite.
- Use Canvas for the editor surface.
- Keep `CadDocument` as the internal source of truth.
- Treat DWG as server-converted, not browser-parsed.
- Use a workspace model with tabs, recent files, and per-tab history.
- Reference copy anchors to external snap points instead of the copied object itself.

## Issues Resolved

- DXF curves initially displayed as chords; LWPOLYLINE bulges and SPLINE sampling were improved.
- Imported lineweights rendered too thick; mapping and stroke caps/joins were refined.
- Large drawings caused interaction lag; snap candidate work and autosave were capped.
- Drag marquee selection conflicted with oversized hitboxes; rectangle/circle selection now targets outlines.
- Text entry did not reliably open; text editor creation was moved to post-click focus handling.

## Issues Deferred

- Production DWG conversion backend integration.
- Full fidelity for all advanced DXF/DWG entities.
- Deeper automated UI/browser testing coverage.

## Technical Debt

- Phase 1-6 implementation was captured mostly through commits and STATE updates, not full per-phase GSD plan/verification artifacts.
