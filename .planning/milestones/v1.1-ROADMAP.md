# Milestone v1.1: File Fidelity and Editing Productivity

**Status:** SHIPPED 2026-04-26
**Phases:** 8-11
**Total Plans:** 15

## Overview

v1.1 strengthened SimpleCAD around real-world CAD file trust, repeated editing productivity, save-state clarity, and automated verification. It builds on the v1.0 editor by improving DXF/DWG handling, adding transform tools, adding Save/Save As semantics, and introducing browser/CLI regression checks.

## Phases

**Plans:** 3 plans

Plans:

- [x] 08-01: DXF round-trip fixtures
- [x] 08-02: Structured warning UI
- [x] 08-03: DWG mode configuration

Delivered:

**Depends on:** Phase 8

## Milestone Summary

**Key Decisions:**

- Keep DWG as server-backed and make mock/server mode explicit.
- Treat DXF fidelity as normalized model comparison rather than raw text comparison.
- Use grouped entities internally while flattening groups for external export compatibility.
- Store File System Access handles only in runtime tab state.
- Use Playwright for canvas workflow coverage and CLI scripts for faster file/performance regression checks.

**Issues Resolved:**

- Reduced ambiguity around DXF curve approximation and unsupported entities.
- Added transform tools needed for repeated editing work.
- Clarified Save vs Save As behavior and unsaved tab state.
- Added repeatable verification commands for workflows, fidelity, performance, and conversion failures.

**Issues Deferred:**

- Production DWG conversion backend.
- Full fidelity for advanced DXF/DWG entity families.
- CI wiring and cross-browser Playwright matrix.
- Mobile/tablet interaction model.

**Technical Debt:**

- E2E is Chromium-only until CI/cross-browser requirements are defined.
- Performance thresholds are local and conservative; tune once CI hardware is known.
