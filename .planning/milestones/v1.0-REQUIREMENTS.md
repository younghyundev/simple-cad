# Requirements Archive: SimpleCAD v1.0

**Archived:** 2026-04-26
**Status:** SHIPPED

## Core Value

사용자가 브라우저에서 2D 도면을 정확하게 보고, 선택하고, 수정하고, DXF/DWG를 포함한 실무 파일 형식으로 다시 저장할 수 있어야 합니다.

## Completed v1 Requirements

### Editor

- [x] **EDIT-01**: 사용자는 랜딩 페이지 없이 전체 화면 CAD 편집기를 바로 볼 수 있다.
- [x] **EDIT-02**: 사용자는 상단 툴바, 왼쪽 도구 패널, 중앙 캔버스, 오른쪽 속성 패널, 하단 상태 바를 사용할 수 있다.
- [x] **EDIT-03**: 사용자는 마우스 좌표, 줌 비율, 선택 상태를 실시간으로 확인할 수 있다.

### Canvas

- [x] **CANV-01**: 사용자는 그리드가 표시된 캔버스에서 도면을 볼 수 있다.
- [x] **CANV-02**: 사용자는 마우스 휠로 확대/축소할 수 있다.
- [x] **CANV-03**: 사용자는 화면을 팬 이동할 수 있다.
- [x] **CANV-04**: 줌과 팬 상태에서도 선택과 편집 좌표가 정확하게 동작한다.

### Entities

- [x] **ENTY-01**: 사용자는 선을 생성, 선택, 이동, 삭제할 수 있다.
- [x] **ENTY-02**: 사용자는 사각형을 생성, 선택, 이동, 삭제할 수 있다.
- [x] **ENTY-03**: 사용자는 원을 생성, 선택, 이동, 삭제할 수 있다.
- [x] **ENTY-04**: 사용자는 폴리라인을 생성, 선택, 이동, 삭제할 수 있다.
- [x] **ENTY-05**: 사용자는 텍스트를 생성하고 내용을 수정할 수 있다.
- [x] **ENTY-06**: 사용자는 선택 객체의 색상, 선 두께, 선 스타일을 변경할 수 있다.
- [x] **ENTY-07**: 사용자는 undo/redo로 편집 작업을 되돌리고 다시 적용할 수 있다.

### Layers

- [x] **LAYR-01**: 사용자는 기본 레이어를 사용할 수 있다.
- [x] **LAYR-02**: 사용자는 레이어를 추가하고 이름을 변경할 수 있다.
- [x] **LAYR-03**: 사용자는 레이어 표시/숨김과 잠금/해제를 제어할 수 있다.
- [x] **LAYR-04**: 사용자는 객체를 특정 레이어에 배치할 수 있다.

### Files

- [x] **FILE-01**: 사용자는 도면을 내부 JSON 형식으로 저장하고 다시 불러올 수 있다.
- [x] **FILE-02**: 사용자는 도면을 SVG로 내보낼 수 있다.
- [x] **FILE-03**: 사용자는 DXF 파일을 가져와 내부 CadDocument로 변환할 수 있다.
- [x] **FILE-04**: 사용자는 현재 도면을 DXF로 내보낼 수 있다.
- [x] **FILE-05**: 사용자는 DWG 가져오기/내보내기/저장 기능의 서버 변환 API 경로를 사용할 수 있다.
- [x] **FILE-06**: 사용자는 지원하지 않는 엔티티와 변환 경고를 확인할 수 있다.
- [x] **FILE-07**: 앱은 자동 저장을 제공한다.

### Architecture

- [x] **ARCH-01**: 앱은 CadDocument 내부 모델을 편집의 단일 진실 소스로 사용한다.
- [x] **ARCH-02**: 파일 처리는 FileManager, ImportService, ExportService, CadModelMapper, ConversionApiClient로 분리된다.
- [x] **ARCH-03**: DWG 변환 엔진은 어댑터 패턴으로 교체 가능해야 한다.

## Notes

- DWG support is complete as an integration path and adapter/API structure. Production use requires connecting a real DWG conversion backend.
- Advanced editing items originally listed under v2, such as dimensions, resize handles, snapping, recent files, and block/reference import improvements, were partially or fully pulled into v1.0 during implementation.
