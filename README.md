# Web CAD Prompt

## 목적

간단한 2D 작업물만 수정할 수 있는 웹 기반 CAD 편집기를 만들기 위한 상세 프롬프트입니다.

목표는 AutoCAD 같은 전문 CAD가 아니라, 웹에서 가벼운 2D 도면을 불러오고 수정하고 저장할 수 있는 실용적인 도구입니다. 사용자는 복잡한 3D 모델링이나 고급 엔지니어링 기능 없이, 선, 사각형, 원, 텍스트, 치수선 같은 기본 2D 요소를 편집할 수 있어야 합니다.

## 기본 프롬프트

```text
간단한 2D 작업물만 수정할 수 있는 웹 기반 CAD 편집기를 만들어줘.

목표는 AutoCAD 같은 전문 CAD가 아니라, 웹에서 가벼운 2D 도면을 불러오고 수정하고 저장할 수 있는 실용적인 도구다. 사용자는 복잡한 3D 모델링이나 고급 엔지니어링 기능 없이, 선, 사각형, 원, 텍스트, 치수선 같은 기본 2D 요소를 편집할 수 있어야 한다.

핵심 요구사항:

1. 기본 화면 구성
- 전체 화면 웹앱 형태로 만든다.
- 상단에는 파일, 실행 취소, 다시 실행, 확대/축소, 저장 버튼을 둔다.
- 왼쪽에는 도구 패널을 둔다.
- 중앙에는 도면을 편집하는 캔버스를 둔다.
- 오른쪽에는 선택한 객체의 속성 패널을 둔다.
- 하단에는 현재 좌표, 줌 비율, 선택 상태를 표시한다.
- 랜딩 페이지가 아니라 바로 편집 화면이 첫 화면이어야 한다.

2. 지원하는 도구
- 선택 도구
- 이동 도구
- 선 그리기
- 사각형 그리기
- 원 그리기
- 폴리라인 그리기
- 텍스트 추가
- 치수선 추가
- 지우개 또는 삭제
- 확대/축소
- 화면 이동
- 스냅 켜기/끄기
- 그리드 켜기/끄기

3. 객체 편집 기능
- 객체 선택
- 여러 객체 선택
- 드래그로 이동
- 크기 조절
- 회전
- 복사
- 삭제
- 실행 취소 / 다시 실행
- 선택한 객체의 색상 변경
- 선 두께 변경
- 선 스타일 변경: 실선, 점선
- 레이어 변경
- 텍스트 내용 수정
- 텍스트 크기 수정

4. 캔버스 기능
- 무한 캔버스처럼 보이게 한다.
- 마우스 휠로 확대/축소한다.
- 마우스 드래그로 화면 이동을 지원한다.
- 배경에는 그리드를 표시한다.
- 스냅 기능을 지원한다.
- 객체의 끝점, 중심점, 교차점에 스냅할 수 있게 한다.
- 현재 마우스 좌표를 실시간으로 표시한다.
- 줌 상태에서도 객체 선택과 편집이 정확해야 한다.

5. 파일 기능
- JSON 형식으로 도면 데이터를 저장할 수 있게 한다.
- 저장한 JSON 파일을 다시 불러올 수 있게 한다.
- SVG로 내보내기 기능을 제공한다.
- DXF, DWG 파일 Import, Export, Save 기능을 포함한다.
- 브라우저 로컬 스토리지 또는 IndexedDB에 자동 저장한다.

6. 레이어 기능
- 기본 레이어를 제공한다.
- 새 레이어 추가
- 레이어 이름 수정
- 레이어 표시/숨김
- 레이어 잠금/해제
- 레이어별 색상 지정
- 객체를 특정 레이어에 배치

7. UI/UX 방향
- 전문 CAD처럼 너무 복잡하게 만들지 말고, Figma나 간단한 도면 편집기처럼 직관적으로 만든다.
- 버튼에는 가능한 경우 lucide-react 아이콘을 사용한다.
- 도구 버튼에는 툴팁을 제공한다.
- 선택된 도구는 명확하게 강조한다.
- 객체를 선택하면 바운딩 박스와 조절 핸들을 보여준다.
- 오른쪽 속성 패널은 선택한 객체 유형에 따라 필요한 속성만 보여준다.
- 모바일보다는 데스크톱 브라우저 사용을 우선한다.
- 화면은 깔끔하고 업무용 도구처럼 차분한 느낌이어야 한다.

8. 기술 조건
- React 기반으로 구현한다.
- TypeScript를 사용한다.
- 캔버스는 HTML Canvas 또는 SVG 중 적합한 방식을 선택한다.
- 객체 선택, 이동, 확대/축소, 좌표 변환 구조를 명확하게 설계한다.
- 상태 관리는 복잡하지 않게 구성하되, undo/redo가 가능하도록 히스토리 구조를 둔다.
- 도면 데이터 모델은 확장 가능하게 설계한다.
- DXF/DWG 기능은 단순 다운로드 버튼 수준이 아니라, 내부 CAD 데이터 모델과 양방향 변환되는 구조로 설계한다.
- DXF는 가능한 범위에서 클라이언트 파싱/생성을 지원하고, DWG는 서버 변환 API를 전제로 구현한다.
- DWG 포맷은 브라우저 단독 구현이 현실적이지 않으므로 변환 엔진을 교체 가능한 어댑터 구조로 분리한다.

9. 데이터 모델 예시
각 도형은 다음과 같은 공통 속성을 가진다.
- id
- type
- layerId
- x
- y
- rotation
- strokeColor
- fillColor
- strokeWidth
- visible
- locked

도형별 추가 속성:
- line: x1, y1, x2, y2
- rect: x, y, width, height
- circle: cx, cy, radius
- polyline: points
- text: x, y, content, fontSize
- dimension: startPoint, endPoint, label

10. 구현 우선순위
먼저 다음 기능을 완성한다.
- 캔버스
- 그리드
- 확대/축소
- 화면 이동
- 선, 사각형, 원, 텍스트 생성
- 객체 선택
- 이동
- 삭제
- 속성 변경
- undo/redo
- JSON 저장/불러오기
- DXF Import/Export 기본 구조

그 다음 기능으로 확장한다.
- 레이어
- 스냅
- 치수선
- SVG 내보내기
- DWG 서버 변환 연동
- 대용량 파일 비동기 변환

11. 품질 기준
- 단순한 데모가 아니라 실제로 조작 가능한 편집기여야 한다.
- 객체를 생성하고 선택하고 수정하고 저장하는 전체 흐름이 동작해야 한다.
- 좌표 변환, 줌, 팬 상태에서도 마우스 입력이 정확해야 한다.
- UI 요소가 서로 겹치지 않아야 한다.
- 작은 화면에서도 최소한 깨지지 않도록 반응형 처리를 한다.
- 코드 구조는 유지보수하기 쉽게 컴포넌트와 유틸 함수로 나눈다.

최종 결과물은 바로 실행 가능한 웹앱이어야 한다.
필요하다면 샘플 도면 데이터를 포함해서 처음 실행했을 때 예시 객체 몇 개가 보이게 해줘.
```

## DXF / DWG Import, Export, Save 요구사항

```text
추가 요구사항: DXF / DWG 파일 Import, Export, Save 기능

이 웹 CAD는 JSON 저장뿐 아니라 DXF, DWG 파일도 다룰 수 있어야 한다.

1. 지원 파일 형식
- JSON: 앱 내부 도면 데이터 저장용 기본 포맷
- SVG: 웹 공유 및 미리보기용 내보내기 포맷
- DXF: 2D CAD 교환 포맷으로 가져오기/내보내기 지원
- DWG: 실제 CAD 파일 호환을 위해 가져오기/내보내기/저장 기능 구조를 포함

2. Import 기능
- 사용자가 `.json`, `.dxf`, `.dwg` 파일을 업로드할 수 있어야 한다.
- JSON 파일은 클라이언트에서 바로 파싱한다.
- DXF 파일은 클라이언트에서 파싱 가능한 라이브러리를 사용하거나, 서버 API를 통해 내부 도면 데이터 모델로 변환한다.
- DWG 파일은 브라우저에서 직접 파싱하기 어렵기 때문에 서버 변환 API를 사용한다.
- Import된 CAD 객체는 앱 내부 데이터 모델로 변환되어 캔버스에 표시되어야 한다.
- DXF/DWG에서 최소한 다음 2D 엔티티를 지원한다:
  - LINE
  - LWPOLYLINE / POLYLINE
  - CIRCLE
  - ARC
  - TEXT / MTEXT
  - DIMENSION
  - INSERT는 초기 버전에서는 단순 그룹 또는 미지원 객체로 처리 가능
- 지원하지 않는 엔티티는 무시하지 말고 "미지원 객체"로 기록하고 사용자에게 알려준다.
- Import 후 레이어 정보, 색상, 선 두께, 선 스타일을 가능한 한 유지한다.

3. Export 기능
- 현재 편집 중인 도면을 `.json`, `.svg`, `.dxf`, `.dwg` 형식으로 내보낼 수 있어야 한다.
- JSON Export는 앱 내부 데이터 모델을 그대로 저장한다.
- SVG Export는 화면 표시용 벡터 파일로 저장한다.
- DXF Export는 내부 도면 객체를 DXF 엔티티로 변환한다.
- DWG Export는 서버 API를 통해 DXF 또는 내부 모델을 DWG로 변환해서 제공한다.
- Export 시 레이어, 색상, 선 두께, 텍스트, 치수선을 가능한 한 보존한다.
- 변환 과정에서 손실되는 정보가 있으면 사용자에게 알려준다.

4. Save 기능
- "저장" 버튼은 현재 열려 있는 파일 형식과 상태에 따라 동작한다.
- 새 도면이면 기본적으로 JSON으로 저장한다.
- 사용자가 DXF 파일을 열었다면 Save 시 DXF로 저장할 수 있어야 한다.
- 사용자가 DWG 파일을 열었다면 Save 시 서버 변환을 통해 DWG로 저장할 수 있어야 한다.
- "다른 이름으로 저장" 기능을 제공한다.
- 저장 가능한 형식:
  - JSON
  - DXF
  - DWG
  - SVG
- 브라우저에서는 File System Access API를 사용할 수 있으면 기존 파일에 덮어쓰기를 지원한다.
- File System Access API를 사용할 수 없는 브라우저에서는 다운로드 방식으로 저장한다.
- 로컬 스토리지 또는 IndexedDB에 자동 저장 기능을 제공한다.

5. 파일 변환 아키텍처
- 파일 처리는 다음 계층으로 분리한다:
  - FileManager: 파일 열기, 저장, 다른 이름으로 저장, 다운로드 처리
  - ImportService: JSON, DXF, DWG 파일을 내부 모델로 변환
  - ExportService: 내부 모델을 JSON, SVG, DXF, DWG로 변환
  - CadModelMapper: 외부 CAD 엔티티와 내부 도형 모델 간 매핑
  - ConversionApiClient: DWG 변환을 위한 서버 API 클라이언트
- DXF는 가능하면 클라이언트에서 처리하되, 복잡한 파일은 서버 변환을 fallback으로 사용한다.
- DWG는 라이선스와 포맷 제약이 있으므로 서버 변환 방식으로 구현한다.

6. 서버 변환 API 요구사항
DWG 처리를 위해 서버 API를 준비한다.

필요 API:
- `POST /api/cad/import`
  - 입력: `.dxf` 또는 `.dwg` 파일
  - 출력: 앱 내부 JSON 도면 모델
- `POST /api/cad/export`
  - 입력: 앱 내부 JSON 도면 모델과 목표 포맷 `dxf | dwg | svg`
  - 출력: 변환된 파일 다운로드 URL 또는 Blob
- `POST /api/cad/validate`
  - 입력: CAD 파일
  - 출력: 지원 가능 여부, 엔티티 목록, 변환 경고
- `GET /api/cad/jobs/:jobId`
  - 대용량 변환 작업 상태 확인

7. DWG 처리 방식
- DWG는 오픈 포맷이 아니므로 프론트엔드에서 직접 구현하지 않는다.
- 다음 중 하나의 방식을 사용할 수 있도록 구조를 만든다:
  - ODA File Converter
  - Autodesk Platform Services
  - LibreDWG 기반 변환 서버
  - 상용 CAD 변환 SDK
- 실제 구현에서는 변환 엔진을 교체할 수 있도록 어댑터 패턴으로 설계한다.
- 변환 실패, 라이선스 제한, 지원하지 않는 DWG 버전 오류를 사용자에게 명확히 표시한다.

8. Import/Export UI
- 상단 파일 메뉴에 다음 항목을 둔다:
  - 새 도면
  - 열기
  - 최근 파일
  - 저장
  - 다른 이름으로 저장
  - 내보내기
- 열기에서 지원 확장자:
  - `.json`
  - `.dxf`
  - `.dwg`
- 다른 이름으로 저장에서 선택 가능한 포맷:
  - JSON
  - DXF
  - DWG
  - SVG
- 파일 변환 중에는 진행 상태를 보여준다.
- 변환 경고가 있으면 완료 후 경고 목록을 보여준다.
- 지원하지 않는 객체가 있으면 "일부 객체가 변환되지 않았습니다"라고 알려준다.

9. 내부 도면 모델에 파일 메타데이터 추가
도면 데이터에는 다음 메타데이터를 포함한다.

type CadDocument = {
  id: string;
  name: string;
  sourceFile?: {
    name: string;
    type: 'json' | 'dxf' | 'dwg' | 'svg';
    lastSavedAt?: string;
    fileHandleAvailable?: boolean;
  };
  units: 'mm' | 'cm' | 'm' | 'inch';
  layers: CadLayer[];
  entities: CadEntity[];
  unsupportedEntities?: UnsupportedCadEntity[];
  importWarnings?: CadWarning[];
};

10. 변환 품질 기준
- DXF/DWG를 가져온 뒤 기본 2D 도형은 시각적으로 원본과 최대한 비슷해야 한다.
- Import 후 사용자가 객체를 선택, 이동, 삭제, 색상 변경, 저장할 수 있어야 한다.
- DXF로 다시 저장했을 때 주요 2D 객체와 레이어 정보가 유지되어야 한다.
- DWG 저장은 서버 변환 결과를 기준으로 제공한다.
- 변환 과정에서 손실된 속성은 경고로 기록한다.
- 대용량 파일의 경우 UI가 멈추지 않도록 비동기 처리한다.
```

## 현실적인 개발 범위 메모

초기 버전에서는 JSON 저장/불러오기와 DXF Import/Export를 먼저 구현하고, DWG는 서버 변환 API 인터페이스와 어댑터 구조부터 준비하는 것이 현실적입니다. DWG는 브라우저에서 직접 안정적으로 처리하기 어렵기 때문에 변환 엔진 선택과 라이선스 검토가 별도 작업으로 필요합니다.
